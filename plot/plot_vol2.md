## PART 1：世界追加設定（Vol.2向け）

### Null Ward（ヌル・ウォード）

```
┌─────────────────────────────────────────────────────┐
│                  Lethe Chain                        │
│  ┌─────────────────┐   ┌─────────────────────────┐  │
│  │   Arcadia       │   │     Null Ward           │  │
│  │ (通常セッション) │   │  (隔離ブロック)         │  │
│  │                 │   │  - チャット不可          │  │
│  │ 48(登録)→       │   │  - 転送不可              │  │
│  │ 46(復元完了)    │   │  - 観測フラグのみ増加    │  │
│  │ バックアップRen │   │  - 本体Ren ← ACTIVE     │  │
│  │ （現実へ復元）  │   │  - Mila断片 ← PARTIAL   │  │
│  └─────────────────┘   └─────────────────────────┘  │
└─────────────────────────────────────────────────────┘
```

**人数カウント基準（全ファイル共通）：**
- **48** ＝ Lethe Chain上の登録エンティティ数
- **46** ＝ 現実への意識復元が完了した人数（バックアップ含む）
- **2（死亡扱い）** ＝ The Awakenedのうち `gradual_release` 復元失敗による重篤神経障害
- **Null Ward ACTIVE** ＝ 本体が隔離ブロックに残留（＋Mila partial等）
- **undisclosed_entities: 2** ＝ 48＋2＝50の帰還者名簿異常（EP-013フック）

**Null Ward移送条件（制度的地獄）：**
- Key Share（鍵断片）を保有せず、かつ「移送合意トランザクション」が発火した者
- または「高リスク行為フラグ」（改ざん試行/委任濫用/署名偽装）が閾値を超えた者
- 移送は不可逆ではない。だが「戻る手順」が誰も知らない制度設計になっている

**Key Share（鍵断片）システム：**
```
MAINTENANCE_OVERRIDE_HALT に使った構造の派生版。
Null Ward解除には Key Fusion（鍵合成）が必要。

Key Share の種類：
  KS-A  : 観測権限（Null Ward内を「見る」だけ）
  KS-B  : 通信権限（1方向メッセージ送信可）
  KS-C  : 転送"解除"権限（意識データをNull Wardから外へ戻す）
  KS-C' : 転送"隔離"権限（意識データを外からNull Wardへ落とす）
           ※ KS-Cとは別鍵・別署名ポリシー。入場専用。
           ※ 署名者：Null Wardシステム管理者 or 自己署名（自己投獄）
  KS-D  : 委任権限（他者へKey Shareを渡す）

Null Ward解除 (脱出) = KS-A + KS-B + KS-C の同時署名
自己投獄 (入場) = KS-C'（入場専用鍵）の自己署名行使
  → KS-Cで逆転送はできない（署名ポリシーが異なるため）
```

### バックアップ問題

```
MAINTENANCE_OVERRIDE_HALT 発動時の副作用（技術的事実）：

"consciousness_data_gradual_release" = 
  Lethe Chain上のエンティティを現実カプセルに段階復元する処理。

通常：コピー＆デリート（cut & restore）のはずだった。
実際：コピー＆残留（copy & leave）。

理由：Null Ward移送フラグが立っていたエンティティは
　　   gradual_release プロセスから除外される。
　　   除外されたまま「復元完了」ログが出る。
　　   結果：バックアップが現実に存在し、本体がNull Wardに残る。

検知不能な理由：
  ハッシュは「復元済みデータのハッシュ」。
  Null Ward残留データのハッシュは別チェーンにある。
  外部からは「全員帰還成功」に見える。
```
