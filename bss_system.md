## BSS-001: 概要

| 項目 | 値 |
|------|----|
| 正式名称 | Babel Station System（BSS） |
| 設計目的 | テスター間のコミュニケーション、情報共有、取引 |
| アクセス方法 | VR内の任意の「BSS端末」（掲示板パネル型オブジェクト。主要都市に設置） |
| UIモデル | 匿名画像掲示板風（スレッド式）＋カテゴリチャンネル |
| 言語 | リアルタイム自動翻訳（`RULE-004`）対応。母語で読み書き可能 |
| 最大同時閲覧 | 制限なし（全テスター同時アクセス可能） |
| 投稿上限 | 1テスターあたり 1分間に3投稿まで（スパム防止） |

## BSS-003: ニュースフィード流入経路

### 仕組み

| 項目 | 詳細 |
|------|----|
| 情報源 | Arcadiaプラットフォームの **外部ニュースクローラ**（NFC: News Feed Crawler） |
| 設計目的 | βテスト中のテスターに世界のニュースを届ける（テスト期間中の孤立感軽減） |
| 動作方式 | NFCはLethe Chain外部のNH社インフラサーバー上で稼働し、RSSフィード・ニュースAPIを巡回 → 要約を生成 → Lethe Chainのステートに書き込み → BSS `#news` チャンネルに自動投稿 |
| 更新頻度 | 6時間ごと（1日4回） |
| フィルタリング | NH社が設定したキーワードフィルタ。元々は「ネガティブすぎるニュース」を除外する緩いフィルタ |

### 事件後のニュースフィード

| 時期 | 状況 |
|------|------|
| Day 0-1 | ニュースは通常通り配信。事件はまだ報道されていない |
| Day 2 | NH社事件が世界的ニュースに。NFCが **事件に関するニュースを配信** → テスターがパニック |
| Day 3 | NH社がNFCのフィルタを強化しようとするが、**ガバナンス凍結でフィルタ設定の変更にもマルチシグが必要**（`GOV-003`のスマートコントラクト設計の副作用）→ フィルタ変更不能 |
| Day 4以降 | 事件関連のニュースが無検閲で配信され続ける。テスターは現実世界の動きをリアルタイムで（6時間遅れで）知ることができる |

> **RULE-070**: NFCは自動運転であり、NH社が意図的に情報を操作することは**構造的に不可能**（ガバナンス凍結の副次効果）。テスターは「検閲されていないニュース」を受け取り続ける。

### 改ざん・欠落の可能性

| リスク | 確率 | 影響 |
|--------|------|------|
| NFC自体の停止（NH社がサーバーを落とす） | 中 | ニュースが来なくなる → テスターの不安増大。ただしNH社は「テスターとの唯一の間接的コミュニケーション手段」として維持する動機がある |
| 外部ニュースソースの偏り | 高 | NFCが巡回するRSSフィードの選定バイアス。英語圏メディアが中心 |
| テスターによるフェイクニュース投稿（#anonymous） | 高 | NFCの自動投稿と区別がつかないようにフォーマットを模倣 → 混乱 |
| NFC生成要約の不正確さ | 中 | 自動要約のため、ニュアンスが失われたり誤解を招く表現になる可能性 |

```
│          │  喪失を認める。各国政府が調査開始。            │
│          │                                             │
│          │  ─────────────────────────────             │
│          │  [入力欄                      ] [送信]       │
└──────────┴───────────────────────────────────────────┘
```

### UI要素

| 要素 | 説明 |
|------|----|
| チャンネル一覧 | 左サイドバー。参加チャンネルのみ表示。未読バッジあり |
| 投稿 | ユーザー名 + タイムスタンプ + 本文。画像添付不可（テキストのみ） |
| 自動翻訳 | 投稿は自動翻訳されるが、原文を表示するトグルあり |
| 検索 | キーワード検索。全チャンネル横断 |
| NFC投稿 | 黄色バッジ + 📰 アイコンで通常投稿と視覚的に区別 |
| スレッド | 投稿への返信はスレッド形式（タイムライン汚染防止） |
| ピン留め | モデレーターが重要投稿をピン留め可能 |
| ブロック | テスター個人が特定ユーザーの投稿を非表示にできる |

## BSS-005: モデレーション

### 事件前

- NH社の運営チームが最終的なモデレーション権限を持つ
- 自動フィルタ（暴言、脅迫、PII検出）が稼働
- 違反投稿は通報 → 運営承認で削除

### 事件後

| 項目 | 状況 |
|------|----|
| 運営権限 | **不在**。運営のモデレーションアカウントは外部アクセス不能 |
| 自動フィルタ | **稼働中**。暴言と脅迫の自動検出は機能しているが、文脈理解が不十分 |
| ボランティアモデレーター | Gazette (`CHAR-011`) を中心に約20名が自主的にモデレーション |
| 権限 | ボランティアモデレーターは「投稿の一時非表示」が可能だが「永久削除」は不可（運営権限のみ） |
| 課題 | (1) フェイクニュースの判別が困難 (2) 派閥間の煽動投稿の対処 (3) #anonymous の無法地帯化 |

### モデレーション上の物語的葛藤

| 状況 | Gazette の選択 |
|------|--------------|
| Exodus の計画詳細が #general に漏洩 | 非表示にするか？（情報の公共性 vs 作戦の安全性） |
| Berserkers が他派閥への攻撃予告 | 非表示にするか？（脅迫 vs 警告情報としての価値） |
| The Awakened の「帰還不要」キャンペーン | 非表示にするか？（信教の自由 vs 帰還妨害の煽動） |
| 外部ニュースに「テスターは既に脳死」という誤報 | 非表示にするか？（パニック防止 vs 情報の自由） |

## OPEN_QUESTION

- **OQ-BSS-001**: BSSにダイレクトメッセージ（DM）機能はあるか？
  - 代替案A: あり（テスター間の私的通信が可能。陰謀や裏取引の温床）
  - 代替案B: なし（すべて公開チャンネルのみ。プライバシーはVR内の対面会話で確保）
  - **推奨**: 代替案A（DMありのほうが物語の幅が広がる）

- **OQ-BSS-002**: NFCが止まった場合のストーリー展開
  - 代替案A: NFCは物語中ずっと稼働（ニュースは途切れない）
  - 代替案B: Day 15頃にNH社がNFCサーバーを意図的に停止（→ テスターの孤立感演出）
  - 代替案C: NFCサーバーが物理的故障で停止（偶発的）
  - **推奨**: 代替案A（ニュース継続が派閥対立を駆動する燃料になるため）
