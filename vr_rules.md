## 物理法則

### RULE-020: 重力

| 項目 | 値 |
|------|----|
| 重力加速度 | 9.8 m/s²（地球準拠） |
| 例外 | Ethereal Gardens（浮遊島群）では 3.0 m/s²（低重力） |
| 水中 | 浮力は現実準拠。Azure Depths では水圧表現あり |

### RULE-021: 移動速度

| 移動手段 | 速度 |
|---------|------|
| 徒歩 | 5 km/h |
| 走行 | 15 km/h |
| 騎乗（マウント） | 40 km/h |
| テレポート | Central Nexusの転移門間のみ。クールダウン30分 |
| 飛行 | Ethereal Gardensのみ可能（低重力エリア限定） |

### RULE-022: 物体の物理

- 物体の衝突、破壊、流体はゲームエンジン準拠（完全な物理シミュレーションではない）
- 破壊された構造物は **24時間（VR内時間）** で再生成（リスポーン）
- テスターが建築した構造物は再生成されない（永続的。ただし破壊は可能）

## 死とリスポーン

### RULE-023: テスターの「死」

| 項目 | 値 |
|------|----|
| HP | 各テスターに HP 1,000 が設定 |
| HPがゼロ | **「死亡」ではなく「気絶」** 扱い |
| 気絶時の挙動 | 画面暗転 → 10分後に最寄りのセーフゾーンでリスポーン |
| 気絶中の意識 | 意識は維持される。暗闇の中で考え事ができる |
| ペナルティ | 所持アイテムの10%がランダムドロップ（気絶地点に散乱） |
| 連続気絶制限 | 1時間以内に3回気絶すると、強制的に30分間セーフゾーンに固定（クールダウン） |

> **RULE-024**: テスターの意識がLethe Chain上に格納されている以上、**VR内で「完全な死」は発生しない**。意識データの消去はガバナンスコマンドでのみ可能であり、通常のゲームプレイでは不可。

> **RULE-025**: ただし、**意識データの破損** は理論的にあり得る。Lethe Chainのストレージ障害や不正なトランザクションにより、スナップショットの `integrity_hash` が不一致になると、そのテスターは「意識の断片化」を経験する可能性がある（伏線として使用可能）。

## 痛覚・疲労・睡眠

### RULE-005: 痛覚（再掲、technology.md より）

- 最大95%減衰。残りは圧迫感程度
- 精神的ストレス・恐怖は減衰対象外（`RULE-006`）

### RULE-030: 疲労

| 項目 | 値 |
|------|----|
| 疲労システム | あり。テスターは活動12時間（VR内時間）で「疲労」ステータス |
| 疲労効果 | 移動速度-20%、反応速度-15%、視界がわずかにぼやける |
| 重度疲労 | 活動20時間で「重度疲労」。移動速度-50%、会話の自動翻訳精度低下 |
| 回復方法 | 睡眠（次項） |

### RULE-031: 睡眠

| 項目 | 値 |
|------|----|
| 睡眠の仕組み | テスターがベッド等の睡眠オブジェクトを使用すると「睡眠モード」に入る |
| 睡眠中 | 意識は薄明状態。夢のような環境が表示される（VR生成）。外部からの呼びかけで覚醒可能 |
| 最低睡眠時間 | 4時間（VR内時間）で疲労回復 |
| 現実の身体 | VR内睡眠中もカプセル内では脳波がシータ波に移行。**実際に脳が休息する**（`TECH-003` の神経I/Oが休息モードに切り替わる） |
| 強制睡眠 | 活動30時間を超えると**強制的に睡眠モード**に突入（安全装置） |

## 時間

### RULE-003: 時間加速（再掲、world_overview.md より）

| 項目 | 値 |
|------|----|
| 加速倍率 | 1.5倍（VR内の体感時間は現実の1.5倍速） |
| 例 | 現実の1日（24h）= VR内の36時間 |
| 昼夜サイクル | VR内で18時間周期（明12h / 暗6h） |
| カレンダー | VR内独自カレンダーは無し。テスターは現実時間を参照可能（RSS経由） |

### RULE-032: 時間加速の制約

- 加速倍率は **固定値** であり、エリアやイベントによる変動はない
- テスター個人が独自の時間速度を持つことはない
- 加速はあくまで「体感」であり、物理演算のタイムステップは一定

## アイテム・通貨

### RULE-040: 通貨

| 項目 | 値 |
|------|----|
| 通貨名 | **Arc（アーク）** |
| 最小単位 | 1 mArc（ミリアーク、0.001 Arc） |
| 初期支給 | テスター1人あたり 10,000 Arc |
| 入手方法 | モンスター討伐報酬、クエスト報酬、クラフト販売、テスター間取引 |
| テスター間取引 | 自由（対面取引、RSS上の取引掲示板） |
| インフレ制御 | モンスター報酬は討伐数に応じて逓減（デフレ圧力） |

### RULE-041: アイテム

| カテゴリ | 例 | 所持上限 |
|---------|----|---------| 
| 武器 | 剣、弓、杖、銃（現代風） | 装備3 + バッグ10 |
| 防具 | 鎧、ローブ、シールド | 装備4 + バッグ5 |
| 消耗品 | 回復ポーション、食料、素材 | バッグ50スタック |
| 素材 | 鉱石、木材、魔石 | バッグ99スタック |
| 特殊 | 転移石、地図、解析ツール | バッグ10 |
| 住居 | 建築モジュール | 配置型、所持不可 |

### RULE-042: アイテムのドロップと喪失

- 気絶時に所持品の10%がドロップ（`RULE-023`）
- ドロップ品は気絶地点に24時間（VR内）の間存続
- 他テスターが拾取可能（→ 盗賊行為の温床になる伏線）

## ログ・監視

### RULE-050: テスター行動ログ

| 項目 | 値 |
|------|----|
| ログ内容 | 位置、アクション（戦闘/取引/会話/建築）、HP変動、アイテム変動 |
| 保存先 | Lethe Chain上のステートストレージ |
| アクセス権限 | テスター本人：自分のログのみ閲覧可能。他者のログは閲覧不可 |
| 管理者権限 | `MAINTENANCE_ADMIN` は全テスターのログを包括的に閲覧可能 |
| ログ改ざん | **不可能**（チェーン上に記録されるため。ただしログの「非表示化」は管理者権限で可能） |

### RULE-051: チャットログ

| 項目 | 値 |
|------|----|
| 記録対象 | テスター間のVR内会話（音声→テキスト変換して保存） |
| プライバシー | 会話相手の同意なしに第三者への開示は不可（NCEのプライバシーフィルタ、`RULE-011`） |
| RSS投稿 | 公開投稿は全テスターが閲覧可能。ログとして永続保存 |

## チート・不正行為

### RULE-060: 禁止行為一覧

| 行為 | 説明 | 罰則 |
|------|------|------|
| バグ悪用 | ゲームバグを利用した不正なアイテム/通貨の増殖 | 24時間の行動制限（移動とRSS閲覧のみ） |
| API不正使用 | 保守API等の本来アクセス不可なエンドポイントへのアクセス | **罰則なし**（βテスト仕様。本来はBAN対象だが、事件後は運営が不在） |
| PK（プレイヤーキル） | 他テスターを故意に気絶させる行為 | PvP指定エリア以外では **赤名マーク** が72時間付与（他テスターに警告表示） |
| 詐欺 | 取引における詐欺行為 | テスター間の自治に委ねられる（運営不在） |
| ハラスメント | 言語的/行動的な嫌がらせ | RSSでの通報 → コミュニティモデレーション |

> **RULE-061**: 事件後、運営がサーバー管理を行えないため、**チート対策の自動システムのみ稼働**。新たなルールの追加や罰則の変更は不可能。テスターは既存のルール枠内で自治する必要がある。

## OPEN_QUESTION

- **OQ-VR-001**: VR内での「食事」の必要性をどう扱うか？
  - 代替案A: 食事不要（栄養はカプセルが供給）。ただしゲーム内で食事するとバフ（能力強化）が付く
  - 代替案B: 食事必須（食べないと空腹デバフ）。生活感の演出になるが管理が煩雑
  - **推奨**: 代替案A（カプセル供給＋バフ方式）

- **OQ-VR-002**: PvP（テスター間戦闘）の頻度と物語上の役割
  - 代替案A: 序盤はほぼなし → 派閥対立で中盤から激化
  - 代替案B: 最初から暴力が横行（世紀末感）
  - **推奨**: 代替案A（段階的激化）
